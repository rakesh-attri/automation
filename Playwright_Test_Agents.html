⚙️ How the Official Playwright Agent Loop Works ▶

The npx playwright init-agents --loop=vscode command generates three pre-built agent files inside .github/agents/. GitHub Copilot reads these as custom agents — each one tells Copilot exactly how to use Playwright's MCP browser tools to explore, write, and fix tests.

🔍
Planner Agent
Reads your task.md. Opens a real browser via MCP. Explores your live Salesforce org. Writes a detailed Markdown test plan.
🎭 planner.agent.md
⚡
Generator Agent
Reads the Planner's .md plan. Produces clean, executable Playwright .spec.js files with assertions and Allure annotations.
🎭 generator.agent.md
🩹
Healer Agent
Runs your test suite. When tests fail, it re-explores Salesforce, identifies broken locators and automatically patches the spec.
🎭 healer.agent.md

This file is auto-generated but you can customise it. It tells VS Code how to start the Playwright MCP server that gives agents real browser control.

  "servers": {
    "playwright": {
      "type": "stdio",
      "command": "npx",
      "args": ["@playwright/mcp@latest"],
      "env": {
        "PLAYWRIGHT_BROWSERS_PATH": "0"
      }
    }
  }
}

1 tests/seed.spec.js — SF Login Fixture▶

The seed file is auto-created by init-agents but you must customise it for Salesforce. Agents copy this fixture into every .spec.js they generate, so all tests start already logged in.

tests/seed.spec.jsjs
import { test, expect } from '@playwright/test';
import 'dotenv/config';
import { waitForSFLoad } from '../utils/sf-helpers.js';

/**
 * SEED SPEC — Salesforce Login Fixture
 * Agents copy this file into every generated test.
 * It handles auth so generated tests start on a logged-in SF page.
 */

// Reuse saved auth state across all tests (set in globalSetup)
test.use({ storageState: './reports/.auth-state.json' });

// Fixture: provides a pre-authenticated SF page to all tests
export const sfTest = test.extend({
  sfPage: async ({ page }, use) => {
    // Navigate to SF Lightning home and wait for shell to load
    await page.goto(process.env.SF_URL);
    await waitForSFLoad(page);

    // Make the page available to the test
    await use(page);
  },
});

// Verify the seed itself works — agents use this as a health check
test('seed: verify Salesforce login works', async ({ page }) => {
  await page.goto(process.env.SF_URL);
  await waitForSFLoad(page);

  // SF Lightning nav bar must be present
  await expect(
    page.locator('[data-id="AppNavigation"], .navContainer')
  ).toBeVisible({ timeout: 20000 });
});
2 memory/framework-memory.md — Persistent Agent Context▶

Agents read this file at the start of every session. It acts as the framework's long-term memory — rules, patterns, and lessons learned.

memory/framework-memory.mdmarkdown
# Framework Memory — SF Agentic Automation
_Updated after each run. Always read before starting._

## Stack
- Playwright Test Agents v1.56+ (official Microsoft)
- VS Code v1.105+ with GitHub Copilot Chat
- Agents: .github/agents/ (Planner, Generator, Healer)
- Auth: globalSetup → reports/.auth-state.json → all tests reuse
- Reports: Allure 3 (HTML) + Playwright HTML + JUnit (CI)

## Non-Negotiable Locator Rules
NEVER use CSS class selectors or XPath with IDs.
ALWAYS use in this priority order:
1. page.getByRole('button', { name: 'New' })
2. page.getByLabel('First Name')
3. page.getByPlaceholder('Search...')
4. page.getByText('Success')
5. page.locator('[aria-label="Close"]')
6. page.locator('.toastMessage')  ← ONLY exception

## SF Lightning Patterns (Always Apply)

### After every navigation or click:
await waitForSFLoad(page);

### Toast assertion (must be immediate — toast auto-dismisses):
const toast = page.locator('.toastMessage');
await expect(toast).toBeVisible({ timeout: 15000 });
await expect(toast).toContainText('was created');

### Modal dialog pattern:
const dialog = page.getByRole('dialog');
await dialog.waitFor({ state: 'visible' });
await dialog.getByLabel('Field Name').fill('value');

### Dropdown / picklist:
await page.getByLabel('Stage').click();
await page.getByRole('option', { name: 'Needs Analysis' }).click();

### List view — always switch from "Recently Viewed":
await page.getByRole('button', { name: /Select a List View/i }).click();
await page.getByRole('option', { name: 'All Leads' }).click();
await waitForSFLoad(page);

## Allure Annotations (required in every generated test)
await allure.epic('CRM');
await allure.feature('Lead Management');
await allure.story('Create New Lead');
await allure.severity('critical');

## File Conventions
- Task files: tasks/[flow-name].md (YOU write)
- Plan files: specs/[flow-name]-plan.md (Planner writes)
- Spec files: tests/[flow-name].spec.js (Generator writes)
- Screenshots: captureScreenshot(page, 'step-name') after every step

## Lessons Learned
- SF spinner must disappear before interacting — waitForSFLoad() handles this
- Timestamp-based test data prevents collision between runs
- List view defaults to "Recently Viewed" — switch to "All [Object]"
- Inline edit requires double-click on field value on detail page
- Account Name lookup: type text → wait 500ms → click autocomplete option

# Agent Context — Quick Reference

## Project Paths
- Tasks (input):  tasks/*.md
- Plans (output): specs/*-plan.md

- URL: process.env.SF_URL

## When Running Planner
1. Read the task file
2. Read memory/framework-memory.md for rules
3. Read memory/sf-selectors.md for known locators
4. Open browser via MCP tools (browser_navigate)
5. Explore the SF org — take screenshots
6. Write detailed plan to specs/[name]-plan.md

## When Running Generator
1. Read specs/[name]-plan.md
2. Read tests/seed.spec.js for the auth fixture pattern
3. Generate tests/[name].spec.js
4. Include: allure annotations, captureScreenshot() at every step,
   expect() assertions at every step, afterEach screenshot on failure

## When Running Healer
1. Run: npx playwright test tests/[name].spec.js
2. For each failure: identify broken locator from error
3. Check memory/sf-selectors.md for alternatives
4. Patch the spec file in place
5. Re-run and repeat until all green

2 utils/sf-helpers.js — Login + Utilities▶
utils/sf-helpers.jsjs
import { chromium } from '@playwright/test';
import fs from 'fs/promises';
import 'dotenv/config';

/** Global Setup — logs into SF once, saves auth for all tests */
export default async function globalSetup() {
  const browser = await chromium.launch({ headless: true });
  const page = await browser.newPage();

  console.log('\n🔐 Logging into Salesforce...');
  try {
    await page.goto(`${process.env.SF_URL}/lightning/page/home`);
    await page.fill('#username', process.env.SF_USERNAME);
    await page.fill('#password', process.env.SF_PASSWORD);
    await page.click('#Login');
    await page.waitForSelector(
      '[data-id="AppNavigation"], .navContainer',
      { timeout: 45000 }
    );
    await fs.mkdir('./reports', { recursive: true });
    await page.context().storageState({ path: './reports/.auth-state.json' });
    console.log('✅ Auth state saved.\n');
  } catch (err) {
    await page.screenshot({ path: './reports/login-failure.png' });
    throw err;
  } finally {
    await browser.close();
  }
}

/** Wait for SF Lightning spinners to clear */
export async function waitForSFLoad(page, timeout = 15000) {
  await page.waitForFunction(
    () => !document.querySelector('.forceListViewManagerSpinner') &&
          !document.querySelector('.slds-spinner_container:not([style*="display: none"])'),
    { timeout }
  ).catch(() => {}); // non-fatal
}

/** Switch list view to "All [Object]" */
export async function switchToAllRecords(page, objectName) {
  try {
    await page.getByRole('button', { name: /Select a List View/i }).click({ timeout: 5000 });
    await page.getByRole('option', { name: `All ${objectName}` }).click();
    await waitForSFLoad(page);
  } catch { /* already on correct view */ }
}
3 utils/reporter-utils.js — Screenshots + Allure▶
utils/reporter-utils.jsjs
import { test } from '@playwright/test';
import * as allure from 'allure-js-commons';

/** Capture screenshot → attach to Playwright report + Allure */
export async function captureScreenshot(page, name = 'screenshot') {
  try {
    const buf = await page.screenshot({ fullPage: true });
    await test.info().attach(name, { body: buf, contentType: 'image/png' });
    await allure.attachment(name, buf, 'image/png');
  } catch (e) {
    console.warn(`Screenshot "${name}" failed:`, e.message);
  }
}

/** Wrap a test action in an Allure step */
export async function sfStep(name, page, fn) {
  return allure.step(name, async () => {
    await fn();
    await captureScreenshot(page, name.toLowerCase().replace(/\s+/g, '-'));
  });
}

3 Jenkinsfile — Declarative Pipeline (add to project root)▶
Jenkinsfilegroovy
pipeline {
  agent any

  /* ── Tools ─────────────────────────────────────────────── */
  tools {
    nodejs 'Node20'         // matches NodeJS plugin global tool name
    allure 'allure'         // matches Allure plugin global tool name
  }

  /* ── Parameters — trigger any task from Jenkins UI ─────── */
  parameters {
    choice(
      name: 'TEST_SUITE',
      choices: ['all', 'lead', 'opportunity'],
      description: 'Which test suite to run'
    )
    booleanParam(
      name: 'HEADED',
      defaultValue: false,
      description: 'Run browser in headed mode (requires display / Xvfb)'
    )
    string(
      name: 'RETRIES',
      defaultValue: '2',
      description: 'Playwright retry count on failure'
    )
  }

  /* ── Environment ────────────────────────────────────────── */
  environment {
    CI          = 'true'
    HEADLESS    = "${!params.HEADED}"
    ALLURE_DIR  = 'reports/allure-results'
    NODE_PATH   = "${WORKSPACE}/node_modules"
  }

  /* ── Options ────────────────────────────────────────────── */
  options {
    timeout(time: 60, unit: 'MINUTES')
    disableConcurrentBuilds()
    buildDiscarder(logRotator(numToKeepStr: '20'))
    timestamps()
  }

  /* ── Triggers ───────────────────────────────────────────── */
  triggers {
    cron('H 2 * * *')         // nightly at ~2 AM
  }

  /* ── Stages ─────────────────────────────────────────────── */
  stages {

    stage('Checkout') {
      steps {
        checkout scm
        sh '''
          echo "Branch: ${GIT_BRANCH}"
          echo "Commit: ${GIT_COMMIT}"
          node --version
          npm --version
        '''
      }
    }

    stage('Install Dependencies') {
      steps {
        sh 'npm ci'
      }
    }

    stage('Install Playwright Browsers') {
      steps {
        sh 'npx playwright install chromium --with-deps'
      }
    }

    stage('Run Tests') {
      steps {
        // Inject SF credentials from Jenkins credential store
        withCredentials([
          string(credentialsId: 'SF_URL',           variable: 'SF_URL'),
          string(credentialsId: 'SF_USERNAME',      variable: 'SF_USERNAME'),
          string(credentialsId: 'SF_PASSWORD',      variable: 'SF_PASSWORD'),
          string(credentialsId: 'SF_SECURITY_TOKEN',variable: 'SF_SECURITY_TOKEN')
        ]) {
          script {
            // Build the right Playwright command based on TEST_SUITE param
            def cmd = 'npx playwright test --config=config/playwright.config.js'
            cmd += " --retries=${params.RETRIES}"

            switch(params.TEST_SUITE) {
              case 'lead':
                cmd += ' tests/lead-creation.spec.js'; break
              case 'opportunity':
                cmd += ' tests/opportunity-flow.spec.js'; break
              default:
                // 'all' — run everything in tests/
            }

            sh cmd
          }
        }
      }
      post {
        always {
          // Stash test results so the Reporting stage always has them
          stash includes: 'reports/**', name: 'test-results', allowEmpty: true
        }
      }
    }

    stage('Generate Reports') {
      steps {
        unstash 'test-results'
        // Allure report from raw results
        sh '''
          allure generate reports/allure-results \
            -o reports/allure-report --clean || true
        '''
      }
    }

  }

  /* ── Post Actions ───────────────────────────────────────── */
  post {
    always {
      // Allure plugin — renders report in the build sidebar
      allure([
        includeProperties: false,
        jdk: '',
        results: [[path: 'reports/allure-results']],
        report: 'reports/allure-report',
        reportBuildPolicy: 'ALWAYS'
      ])

      // JUnit results — shows test trend graph on build page
      junit(
        testResults: 'reports/junit-results.xml',
        allowEmptyResults: true
      )

      // Archive screenshots and traces on failure
      archiveArtifacts(
        artifacts: 'reports/test-results/**,screenshots/**',
        allowEmptyArchive: true
      )

      // Playwright HTML report as archived artifact
      publishHTML(target: [
        allowMissing: true,
        alwaysLinkToLastBuild: true,
        keepAll: true,
        reportDir: 'reports/playwright-report',
        reportFiles: 'index.html',
        reportName: 'Playwright Report'
      ])
    }

    success {
      echo '✅ All SF tests passed!'
    }

    failure {
      echo '❌ Tests failed — check Allure report and archived screenshots.'
    }

    unstable {
      echo '⚠️  Some tests are unstable (flaky). Review the Allure trend.'
    }
  }
}

4 utils/sf-api-helpers.js — Auth + Request Wrapper▶
utils/sf-api-helpers.jsjs
import 'dotenv/config';

/**
 * SF API HELPERS
 * Thin wrapper around Playwright's request fixture for Salesforce REST API.
 * Auto-detects auth: OAuth (CLIENT_ID set) or direct Session ID.
 */

const API_VERSION = process.env.SF_API_VERSION || 'v58.0';

/** Get the Authorization header value — try OAuth first, fall back to Session ID */
export async function getSFToken(request) {
  if (process.env.SF_CLIENT_ID) {
    // Option A: Username-Password OAuth flow
    const res = await request.post(`${process.env.SF_URL}/services/oauth2/token`, {
      form: {
        grant_type: 'password',
        client_id: process.env.SF_CLIENT_ID,
        client_secret: process.env.SF_CLIENT_SECRET,
        username: process.env.SF_USERNAME,
        password: process.env.SF_PASSWORD + (process.env.SF_SECURITY_TOKEN || ''),
      },
    });
    const body = await res.json();
    return `Bearer ${body.access_token}`;
  }
  // Option B: Direct Session ID
  return `Bearer ${process.env.SF_SESSION_ID}`;
}

/** Build the base REST API URL */
export function apiUrl(path) {
  return `${process.env.SF_URL}/services/data/${API_VERSION}/${path}`;
}

/** GET helper with auth injected */
export async function sfGet(request, path, token) {
  return request.get(apiUrl(path), {
    headers: { Authorization: token },
  });
}

/** POST helper */
export async function sfPost(request, path, data, token) {
  return request.post(apiUrl(path), {
    headers: { Authorization: token },
    data,
  });
}

/** PATCH helper (update) */
export async function sfPatch(request, path, data, token) {
  return request.patch(apiUrl(path), {
    headers: { Authorization: token },
    data,
  });
}

/** DELETE helper */
export async function sfDelete(request, path, token) {
  return request.delete(apiUrl(path), {
    headers: { Authorization: token },
  });
}

/** SOQL query helper */
export async function sfQuery(request, soql, token) {
  const q = encodeURIComponent(soql);
  return request.get(apiUrl(`query/?q=${q}`), {
    headers: { Authorization: token },
  });
}
5 tests/API/sf-leads-api.spec.js — Complete Example▶

This is a ready-to-run API test file. It tests Create, Read, Update, Delete and SOQL query on the Lead object — all via Salesforce REST API.

tests/API/sf-leads-api.spec.jsjs
import { test, expect } from '@playwright/test';
import * as allure from 'allure-js-commons';
import {
  getSFToken, sfGet, sfPost, sfPatch, sfDelete, sfQuery
} from '../../utils/sf-api-helpers.js';

/** Shared across all tests in this file */
let token;
let createdLeadId;

test.describe('Salesforce Leads API', () => {

  test.beforeAll(async ({ request }) => {
    token = await getSFToken(request);
  });

  test.beforeEach(async () => {
    await allure.epic('CRM API');
    await allure.feature('Lead REST API');
    await allure.severity('critical');
  });

  // ── CREATE ────────────────────────────────────────────────
  test('POST /sobjects/Lead — create a new Lead', async ({ request }) => {
    await allure.story('Create Lead via API');

    const res = await sfPost(request, 'sobjects/Lead', {
      FirstName: 'API',
      LastName: `Test-${Date.now()}`,
      Company: 'SF Agentic API Inc.',
      Email: 'api.test@sf-framework.com',
      Status: 'Open - Not Contacted',
      LeadSource: 'Web',
    }, token);

    expect(res.status()).toBe(201);

    const body = await res.json();
    expect(body.success).toBe(true);
    expect(body.id).toBeTruthy();

    createdLeadId = body.id; // save for next tests
    await allure.parameter('Created Lead ID', createdLeadId);
  });

  // ── READ ──────────────────────────────────────────────────
  test('GET /sobjects/Lead/:id — read the created Lead', async ({ request }) => {
    await allure.story('Read Lead via API');

    const res = await sfGet(request, `sobjects/Lead/${createdLeadId}`, token);

    expect(res.status()).toBe(200);

    const lead = await res.json();
    expect(lead.FirstName).toBe('API');
    expect(lead.Company).toBe('SF Agentic API Inc.');
    expect(lead.Status).toBe('Open - Not Contacted');
    expect(lead.LeadSource).toBe('Web');
  });

  // ── UPDATE ────────────────────────────────────────────────
  test('PATCH /sobjects/Lead/:id — update Lead status', async ({ request }) => {
    await allure.story('Update Lead via API');

    const res = await sfPatch(request, `sobjects/Lead/${createdLeadId}`, {
      Status: 'Working - Contacted',
      Rating: 'Hot',
    }, token);

    expect(res.status()).toBe(204); // SF returns 204 No Content on PATCH

    // Verify the update took effect
    const check = await sfGet(request, `sobjects/Lead/${createdLeadId}`, token);
    const updated = await check.json();
    expect(updated.Status).toBe('Working - Contacted');
    expect(updated.Rating).toBe('Hot');
  });

  // ── SOQL QUERY ────────────────────────────────────────────
  test('GET /query — SOQL search for the created Lead', async ({ request }) => {
    await allure.story('Query Leads via SOQL');

    const res = await sfQuery(
      request,
      `SELECT Id, FirstName, LastName, Status FROM Lead WHERE Id = '${createdLeadId}'`,
      token
    );

    expect(res.status()).toBe(200);

    const result = await res.json();
    expect(result.totalSize).toBe(1);
    expect(result.records[0].Id).toBe(createdLeadId);
    expect(result.records[0].FirstName).toBe('API');
  });

  // ── DELETE ────────────────────────────────────────────────
  test('DELETE /sobjects/Lead/:id — clean up test Lead', async ({ request }) => {
    await allure.story('Delete Lead via API');

    const res = await sfDelete(request, `sobjects/Lead/${createdLeadId}`, token);

    expect(res.status()).toBe(204);

    // Confirm it's gone — should 404 now
    const gone = await sfGet(request, `sobjects/Lead/${createdLeadId}`, token);
    expect(gone.status()).toBe(404);
  });

import { test, expect } from '@playwright/test';
import * as allure from 'allure-js-commons';
import { getSFToken, sfPost, sfGet, sfDelete } from '../../utils/sf-api-helpers.js';

let token;
let opportunityId;
let accountId;

test.describe('Salesforce Opportunities API', () => {

  test.beforeAll(async ({ request }) => {
    token = await getSFToken(request);

    // Create a test Account to link the Opportunity to
    const acct = await (
      await request.post(`${process.env.SF_URL}/services/data/v58.0/sobjects/Account`, {
        headers: { Authorization: token, 'Content-Type': 'application/json' },
        data: { Name: `API Test Account ${Date.now()}` },
      })
    ).json();
    accountId = acct.id;
  });

  test.afterAll(async ({ request }) => {
    // Clean up Account (Opportunity auto-deletes with it)
    if (accountId) {
      await sfDelete(request, `sobjects/Account/${accountId}`, token);
    }
  });

  test.beforeEach(async () => {
    await allure.epic('CRM API');
    await allure.feature('Opportunity REST API');
  });

  test('POST — create Opportunity and verify fields', async ({ request }) => {
    await allure.story('Create Opportunity via API');

    const closeDate = new Date();
    closeDate.setDate(closeDate.getDate() + 30);

    const res = await sfPost(request, 'sobjects/Opportunity', {
      Name: `API Opp ${Date.now()}`,
      AccountId: accountId,
      StageName: 'Needs Analysis',
      CloseDate: closeDate.toISOString().split('T')[0],
      Amount: 50000,
    }, token);

    expect(res.status()).toBe(201);
    const body = await res.json();
    expect(body.success).toBe(true);
    opportunityId = body.id;

    // Verify the created record
    const check = await sfGet(request, `sobjects/Opportunity/${opportunityId}`, token);
    const opp = await check.json();
    expect(opp.StageName).toBe('Needs Analysis');
    expect(opp.Amount).toBe(50000);
  });

});

Add a parallel API stage to the existing Jenkinsfile so UI and API tests run side-by-side:

stage('Run Tests') {
  parallel {

    stage('UI Tests') {
      when { expression { params.TEST_SUITE == 'all' || params.TEST_SUITE == 'ui' } }
      steps {
        withCredentials([
          string(credentialsId: 'SF_URL',          variable: 'SF_URL'),
          string(credentialsId: 'SF_USERNAME',     variable: 'SF_USERNAME'),
          string(credentialsId: 'SF_PASSWORD',     variable: 'SF_PASSWORD'),
          string(credentialsId: 'SF_SECURITY_TOKEN',variable: 'SF_SECURITY_TOKEN')
        ]) {
          sh """
            npx playwright test \
              --config=config/playwright.config.js \
              --project=ui-tests \
              --retries=${params.RETRIES}
          """
        }
      }
    }

    stage('API Tests') {
      when { expression { params.TEST_SUITE == 'all' || params.TEST_SUITE == 'api' } }
      steps {
        withCredentials([
          string(credentialsId: 'SF_URL',          variable: 'SF_URL'),
          string(credentialsId: 'SF_USERNAME',     variable: 'SF_USERNAME'),
          string(credentialsId: 'SF_PASSWORD',     variable: 'SF_PASSWORD'),
          string(credentialsId: 'SF_SECURITY_TOKEN',variable: 'SF_SECURITY_TOKEN'),
          string(credentialsId: 'SF_SESSION_ID',   variable: 'SF_SESSION_ID')
        ]) {
          sh """
            npx playwright test \
              --config=config/playwright.config.js \
              --project=api-tests \
              --retries=1
          """
        }
      }
    }

  }
}
